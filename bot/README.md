# سرویس ربات تلگرام/بله (Magic Link) — فاز ۲۵

این یک **سرویس Node.js مجزا** از اپ اصلی Next.js است (طبق PRD، برای کاهش بار کانکشن‌های همزمان). نمی‌توان آن را داخل همان پروسه‌ی Next اجرا کرد چون نیاز به long-polling/webhook دائمی دارد.

## چه می‌کند
1. کاربر در تلگرام/بله `/start` می‌زند.
2. ربات `POST /api/bot/magic-link` اپ اصلی را با `Authorization: Bearer $BOT_API_SECRET` صدا می‌زند و شناسه‌ی پیام‌رسان کاربر را می‌فرستد.
3. اپ یک توکن یکبارمصرف (Redis، ۱۵ دقیقه) می‌سازد و **لینک ورود کامل** برمی‌گرداند.
4. ربات لینک را برای کاربر می‌فرستد؛ کاربر کلیک می‌کند → `/api/auth/magic?token=...` کوکی JWT ست می‌کند و وارد می‌شود.

> سمت اپ اصلی همه چیز آماده و تست‌شده است: `lib/magic.ts`، `app/api/bot/magic-link`، `app/api/auth/magic`.
> همچنین مسیر جایگزین «مینی‌اپ» (`/api/auth/miniapp` با `initData` امضاشده) از قبل پیاده شده است.

## راه‌اندازی
```bash
cd bot
npm install
cp .env.example .env   # مقادیر را پر کنید
npm start
```

## متغیرهای محیطی (`bot/.env`)
```env
TELEGRAM_BOT_TOKEN=...      # از BotFather
APP_URL=https://your-app.example.com
BOT_API_SECRET=...          # باید با BOT_API_SECRET اپ اصلی یکی باشد
```

## نوتیفیکیشن خروجی
برای ارسال نوتیف از طرف اپ به ربات (یادآور مطالعه، خطر زنجیره) دو گزینه:
- **Web Push** (پیاده‌شده، فاز ۲۶): داخل خود PWA، بدون نیاز به ربات.
- **از طریق ربات**: یک endpoint داخلی در همین سرویس بسازید که اپ اصلی (از cron) صدا بزند و پیام را با `sendMessage` بفرستد. (اختیاری — وقتی لازم شد اضافه شود.)
